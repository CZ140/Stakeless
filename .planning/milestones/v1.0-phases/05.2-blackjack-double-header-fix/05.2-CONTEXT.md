# Phase 5.2: Blackjack Double Down & Header Profile Link Fix - Context

**Gathered:** 2026-03-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix two specific code defects:
1. The `/blackjack/double` API handler omits `dealerValue` from both response paths (bust and normal completion), causing the frontend to display "?" for the dealer hand total after a double down.
2. The `Header.tsx` `<BalanceDisplay />` is wrapped in a `<Link to="/profile">` that routes to a non-existent page (Phase 7 will build it), silently bouncing users to a 404 or blank route.

No new features, no scope expansion. This phase closes BJK-02, BJK-04, and AUTH-05 traceability gaps.

</domain>

<decisions>
## Implementation Decisions

### Double Down response fix
- Add `dealerValue` only to both `res.json()` calls in the `/blackjack/double` handler — the bust path and the normal-completion path
- Do NOT add `playerValue` — the frontend `DoubleResponse` interface does not consume it, and minimal change is preferred
- Both response sites: `dealerValue: calculateHandValue(state.dealerHand).value`

### Header balance link
- Remove the `<Link to="/profile">` wrapper entirely — `BalanceDisplay` becomes a plain display element with no click behavior
- Do NOT route to `/dashboard` or any stub destination — honest non-clickable is cleaner than misleading navigation
- No `cursor: default` override needed (BalanceDisplay already has no pointer styling)

### Claude's Discretion
- No additional styling or layout changes to Header
- No changes to BalanceDisplay component itself

</decisions>
