## Decisions Made

This is a known minor UX inaccuracy, accepted for v1.0 and documented in RESEARCH.md as Option A. Fixing it would require tracking "has user ever logged in" state, which adds complexity not warranted now.
- Sign Out button scope is limited to pages rendering Header (currently DashboardPage). Phase 4 game pages will need to include Header or a shared layout to surface logout — tracked as tech debt, not fixed here.
