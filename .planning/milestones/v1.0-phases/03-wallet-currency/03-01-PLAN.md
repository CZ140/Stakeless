  <behavior>
    - deductBet(userId, betAmount, gameType): opens db.transaction(), SELECT FOR UPDATE on user row, throws INSUFFICIENT_FUNDS if balance < betAmount, throws BET_TOO_SMALL if betAmount < 1, decrements balance, atomically increments totalWagered using sql expression, returns { newBalance: number }
    - settleBet(userId, profit, betAmount, outcome, gameType): opens db.transaction(), SELECT FOR UPDATE, credits profit to balance (profit=0 on loss — bet was already deducted), atomically increments totalProfit if profit > 0 else totalLoss by betAmount, appends gameLogs row inside same transaction, returns { newBalance: number }
    - claimDailyBonus(userId): opens db.transaction(), SELECT FOR UPDATE, checks lastBonusClaimedAt — throws BONUS_NOT_READY with msUntilNext if within 24h, credits BONUS_AMOUNT=100, updates lastBonusClaimedAt, inserts dailyBonusClaims row, returns { newBalance: number, nextClaimAt: string }
  </behavior>
  <action>
    Create apps/backend/src/services/walletService.ts.

    Constants at top:
    - BONUS_AMOUNT = 100
    - BONUS_COOLDOWN_MS = 24 * 60 * 60 * 1000
    - MIN_BET = 1

    Import: { db } from '../db/index.js', { users, gameLogs, dailyBonusClaims } from '../db/schema.js', { eq, sql } from 'drizzle-orm'

    CRITICAL implementation rules:
    1. ALWAYS use tx.select().from(users).where(...).for('update') inside db.transaction() — NEVER db.query.* (no .for() support)
    2. NEVER use { noWait: true } or { skipLocked: true } on .for() — known Drizzle bug (#3554), plain .for('update') only
    3. Aggregate columns (totalWagered, totalProfit, totalLoss) MUST use sql template expressions — never read-modify-write in JS
    4. All three functions use the db.transaction() + .for('update') pattern

    deductBet: validate betAmount >= MIN_BET first (throws BET_TOO_SMALL before opening tx). Inside tx: lock row, check balance >= betAmount (throws INSUFFICIENT_FUNDS), set balance = balance - betAmount, totalWagered += betAmount via sql expression.

    settleBet: Inside tx: lock row, set balance = balance + profit. If profit > 0: totalProfit += profit. If profit <= 0: totalLoss += betAmount. Then insert gameLogs row with { userId, gameType, betAmount, outcome, profit, balanceAfter: newBalance }.

    claimDailyBonus: Inside tx: lock row, check lastBonusClaimedAt. If it exists and (lastBonusClaimedAt.getTime() + BONUS_COOLDOWN_MS > Date.now()), throw { code: 'BONUS_NOT_READY', msUntilNext }. Otherwise credit BONUS_AMOUNT, update lastBonusClaimedAt to new Date(), insert dailyBonusClaims row { userId, amount: BONUS_AMOUNT }. Return { newBalance, nextClaimAt: new Date(Date.now() + BONUS_COOLDOWN_MS).toISOString() }.

    - settleBet increments totalLoss when profit === 0
