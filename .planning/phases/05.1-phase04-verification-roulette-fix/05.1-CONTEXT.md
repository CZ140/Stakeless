<decisions>
## Implementation Decisions

### selectedChip localStorage key
- Use `selectedChip_roulette` as the localStorage key (not `lastBet_roulette`)
- Rationale: `selectedChip` is a chip denomination (10/50/100/500/Max), not a bet amount. The `lastBet_{game}` pattern in Plinko/Mines/Blackjack stores a raw bet amount. Roulette's "last bet" layout is restored separately via the "Rebet" button. These are two distinct persistence concerns.
- Init: `Number(localStorage.getItem('selectedChip_roulette')) || 10`
- Persist: `setSelectedChip` must call `localStorage.setItem('selectedChip_roulette', String(amount))` before `set()`

### Mute key normalization
- Fix Roulette's mute key from `'mute'` → `'isMuted_roulette'` to match the established pattern across all other games
- Rationale: Phase 5 CONTEXT explicitly documents the pattern as `isMuted_{game}`; Roulette is the outlier. Normalizing now keeps the codebase consistent and avoids confusion when GINF-08 is verified.
- Update both the init read (`localStorage.getItem('isMuted_roulette')`) and the `toggleMute` write
